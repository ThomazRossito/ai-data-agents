## O que aconteceu
Agente `databricks-engineer` entrou em loop de retentativas (>3) após escrita incompleta do arquivo `/ecom` (path truncado, sem extensão) via tool `Write`, sem progresso nas tarefas entre 12:24:52 e 12:27:40.

## Causa raiz
Tool `Write` recebeu path malformado (`/ecom` ao invés do padrão `spec_ecommerce_analytics_databricks.md`), provavelmente por truncamento do nome do arquivo na geração do parâmetro `file_path`, causando falha silenciosa e retentativas sem avanço.

## Padrão para evitar
Sempre validar que `file_path` termina com extensão `.md` e contém nome completo do artefato antes de chamar `Write`; nunca reutilizar prefixos truncados de paths anteriores da sessão.
