## O que aconteceu
Agente `databricks-engineer` entrou em loop de retentativas (>3) após sequência de 3 chamadas `mcp__databricks__execute_sql` em 22 segundos, sem progresso na tarefa.

## Causa raiz
Queries SQL executadas sem validação de schema prévio (`mcp__databricks__list_tables` listou tabelas, mas não houve inspeção de colunas/ tipos antes dos `execute_sql`), provavelmente gerando erros silenciosos ou resultados vazios que dispararam retentativas automáticas.

## Padrão para evitar
Sempre executar `mcp__databricks__describe_table` ou equivalente de schema inspection antes de qualquer `execute_sql` em tabelas não cacheadas; nunca encadear mais de 2 `execute_sql` consecutivos sem validar output intermediário.
