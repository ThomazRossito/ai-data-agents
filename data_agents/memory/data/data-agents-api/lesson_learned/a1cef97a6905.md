## O que aconteceu
Agente `fabric-engineer` travou em loop de retentativas (>3) após erro de digitação no tool call `mcp__contex` (truncamento de `mcp__context` ou nome inexistente), sem validação de tool disponível antes da invocação.

## Causa raiz
Ausência de validação do nome da tool contra o registry de tools disponíveis do MCP server `fabric_community`; erro de truncamento/typo não foi capturado, gerando falha silenciosa e retentativas cegas.

## Padrão para evitar
Sempre validar o nome da tool contra `mcp__list_tools` ou registry local antes de executar; nunca retentar a mesma tool >1 vez sem verificar se o nome existe e se o erro é recuperável.
