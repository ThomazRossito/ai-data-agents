## O que aconteceu
Agente `fabric-engineer` entrou em loop de retentativas (>3) sem progresso após sequência de `TodoWrite` e `Write` para arquivos `.md` de documentação de arquitetura medallion.

## Causa raiz
`TodoWrite` truncado (`TodoW`) indica falha de parsing/timeout na atualização de estado; agente não validou confirmação da tool e repetiu ciclo de escrita sem verificar persistência do todo.

## Padrão para evitar
Sempre verificar `status`/`ack` do `TodoWrite` antes de invocar `Write`; nunca encadear writes de artefatos se o todo list não confirmar atualização com sucesso.
