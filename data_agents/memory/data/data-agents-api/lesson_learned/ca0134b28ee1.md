## O que aconteceu
Agente `fabric-engineer` entrou em loop de retentativas (>3) após execuções sequenciais de `TodoWrite` e `Write` para arquivos de documentação (PRD e specs), sem progresso entre ciclos de 23:07 a 23:12.

## Causa raiz
Falta de validação de estado entre `TodoWrite` e `Write`: o agente reescreveu tarefas sem verificar se o artefato anterior foi concluído ou se o contexto de saída mudou, gerando ciclos vazios.

## Padrão para evitar
Sempre verificar `hash/conteúdo do último artefato escrito` antes de novo `TodoWrite`; nunca reescrever todo list se o arquivo de saída não teve alteração semântica detectada.
