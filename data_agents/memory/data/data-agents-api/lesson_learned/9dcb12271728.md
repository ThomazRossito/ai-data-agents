## O que aconteceu
Agente `fabric-engineer` entrou em loop de retentativas (>3) após chamar `mcp__fabric_notebook__fabric_pipeline_run` e delegar para sub-agente `Agent` sem validar status da execução do pipeline.

## Causa raiz
Sub-agente `Agent` iniciou nova iteração sem verificar se o `fabric_pipeline_run` concluiu, gerando chamadas redundantes e retentativas sem progresso de estado.

## Padrão para evitar
Sempre consultar `fabric_pipeline_run_status` e confirmar estado terminal (`Succeeded`/`Failed`/`Cancelled`) antes de invocar sub-agente `Agent` para continuação de workflow Fabric.
