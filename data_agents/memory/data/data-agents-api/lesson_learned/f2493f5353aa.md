## O que aconteceu
Agente `databricks-engineer` entrou em loop de retentativas (>3) após sequência de 5+ calls às tools `mcp__migration_source__*` sem progresso para o objetivo de migração.

## Causa raiz
Falta de validação de guardrails entre fases de discovery: agente executou `list_sources` → `diagnostics` → `get_schema_summary` → `count_tables_by_schema` → `list_schemas` sem critério de parada nem verificação se dados coletados já eram suficientes para próxima etapa.

## Padrão para evitar
Sempre definir `max_discovery_tools` por fase (ex: 2 calls) e exigir checkpoint de decisão antes de transitar de "coleta de metadados" para "execução de migração"; nunca chamar `list_schemas` após `get_schema_summary` sem validar se schemas já foram identificados.
