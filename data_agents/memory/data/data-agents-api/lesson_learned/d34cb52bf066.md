## O que aconteceu
Agente `databricks-engineer` entrou em loop de retentativas (>3) após sequência de 5+ chamadas a tools `mcp__migration_source__*` sem transição para ação de migração.

## Causa raiz
Falta de validação de saída entre fase de discovery (list/diagnostics/schema) e fase de execução; agente não detectou que já possuía schema summary e continuou coletando metadados redundantes.

## Padrão para evitar
Sempre verificar `schema_summary` retornado antes de chamar `count_tables_by_schema` ou `list_schemas`; nunca executar mais de 2 tools de discovery consecutivas sem decisão de próximo passo.
