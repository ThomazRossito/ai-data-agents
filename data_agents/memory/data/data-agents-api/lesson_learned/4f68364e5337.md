## O que aconteceu
Agente `fabric-engineer` entrou em loop de retentativas (>3) após execuções sequenciais de `TodoWrite` e `Write` para arquivos de documentação (`prd_poc_itausa_fabric_medallion_docx.md`, `spec_poc_itausa_fabric_medallion_docx.md`), sem progresso verificável entre ciclos.

## Causa raiz
Falta de validação de estado entre `TodoWrite` e `Write`: o agente reescreveu tarefas sem verificar se o artefato anterior foi concluído ou se houve mudança real no conteúdo, causando ciclos vazios de "planejar → escrever → replanejar".

## Padrão para evitar
Sempre incluir checkpoint de validação de artefato (`Read` ou hash de conteúdo) entre `Write` e o próximo `TodoWrite`; nunca gerar novo `TodoWrite` se o arquivo de saída não tiver sido modificado desde o ciclo anterior.
