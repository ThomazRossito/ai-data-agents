## O que aconteceu
Agente `databricks-engineer` entrou em loop de retentativas após execução truncada da tool `mcp__databricks__` (nome incompleto no log), sem progresso nas ações subsequentes.

## Causa raiz
Chamada à tool MCP Databricks falhou com nome truncado (`mcp__databricks__` ao invés de `mcp__databricks__<ação>`), indicando parsing incorreto do schema/tool name ou resposta malformada do servidor MCP.

## Padrão para evitar
Sempre validar o nome completo da tool MCP no payload antes de executar; nunca propagar chamadas com sufixo vazio ou truncado.
