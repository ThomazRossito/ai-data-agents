## O que aconteceu
Agente `fabric-engineer` entrou em loop de retentativas (>3) após execuções sequenciais de `TodoWrite` e `Write` sem progresso verificável na geração do artefato final.

## Causa raiz
Falta de validação de estado entre `TodoWrite` e `Write`: o agente marcava tarefas como concluídas sem verificar se o conteúdo escrito satisfazia o PRD gerado anteriormente, causando reescritas vazias.

## Padrão para evitar
Sempre validar conteúdo mínimo esperado (schema, seções obrigatórias) após `Write` antes de chamar `TodoWrite`; nunca marcar todo como done sem diff produtivo entre versões consecutivas do mesmo arquivo.
