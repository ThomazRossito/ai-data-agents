## O que aconteceu
Agente `databricks-engineer` entrou em loop de retentativas (>3) após sequência de 5+ calls de discovery no `mcp__migration_source` sem transição para ação de migração.

## Causa raiz
Discovery ad infinitum: agente não implementa guarda de "informação suficiente" para sair da fase de exploração de schemas/tabelas e iniciar a transformação.

## Padrão para evitar
Sempre definir `max_discovery_calls` (ex: 3) antes de forçar decisão de escopo; nunca chamar `list_schemas` após `get_schema_summary` + `count_tables_by_schema` na mesma sessão.
