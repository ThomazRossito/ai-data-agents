## O que aconteceu
O agente de custos Azure realizou dezenas de chamadas sequenciais à API Azure Retail Prices (list_skus, get_retail_price, estimate_monthly_cost) em curto espaço de tempo, resultando em rate limiting e falhas para obter preços de SKUs específicos.

## Causa raiz
A API Azure Retail Prices não suporta alto volume de requisições sequenciais sem throttling. A abordagem inicial de múltiplas chamadas individuais para diferentes SKUs excedeu os limites de taxa.

## Padrão para evitar
Implementar backoff exponencial entre chamadas, consolidar requisições quando possível, ou usar abordagem mais direta com menos chamadas à API. Considerar cache local de preços consultados frequentemente.
