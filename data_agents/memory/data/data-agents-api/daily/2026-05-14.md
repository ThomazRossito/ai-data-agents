source_session: cli-5edf454a
confidence: 0.95

## O que aconteceu
O pipeline da PoC Itaúsa falhou porque o notebook nb_raw_to_bronze continha `%pip install python-docx`, que está desabilitado no ambiente Microsoft Fabric.

## Causa raiz
O Microsoft Fabric não permite comandos `%pip` em notebooks de pipeline; bibliotecas externas precisam ser pré-instaladas via Environment ou deve-se usar apenas stdlib do Python.

## Padrão para evitar
Nunca usar `%pip install` em notebooks Fabric. Para manipulação de DOCX, usar zipfile + xml.etree.ElementTree (stdlib) ao invés de python-docx.

summary: fabric-engineer: Environment customizado necessário para bibliotecas não-stdlib
tags: fabric-engineer, microsoft-fabric, environment, stdlib, import
source_session: cli-5edf454a
confidence: 0.95

## O que aconteceu
Após remover %pip install python-docx, o código ainda fazia `from docx import Document`, que falhava por biblioteca não instalada. Foi necessário verificar existência de Environment com a biblioteca.

## Causa raiz
Fabric permite bibliotecas externas apenas via Environment configurado no workspace, não via %pip em runtime. O agente não verificou disponibilidade do Environment antes de gerar código com import externo.

## Padrão para evitar
Sempre verificar se biblioteca está na stdlib antes de usar. Se necessário biblioteca externa, confirmar existência de Environment pré-configurado. Preferir stdlib para máxima portabilidade em Fabric.
