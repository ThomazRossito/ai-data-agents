## O que aconteceu
Durante o cálculo de custos para Itausa, o agente descobriu que Azure OpenAI não tinha preços disponíveis na região primária solicitada, forçando uso de get_price_with_regional_fallback.

## Causa raiz
Azure OpenAI tem disponibilidade regional limitada e preços nem sempre publicados em todas as regiões do Azure Pricing API. O fallback automático pode mascarar diferenças de latência/compliance.

## Padrão para evitar
Sempre validar explicitamente regiões disponíveis para Azure OpenAI antes do cálculo; documentar no relatório quando fallback regional é usado para serviços de IA; considerar East US/Sweden Central como primárias para OpenAI.
